To run pipeline successfully, two files are necessary.

One is .py file which includes analysis steps, while config file which allocate necessary files. Both of them can be found in /script/

There're two types of config files which called `configForMainTail.yaml` and `configForModifAS.yaml`.

First is used for dRNAmain and dRNAtail, while other is for dRNAmodif_1/_2 and dRNAas.py.

You needn't to worry since the location of yaml has been assigned in .py files.

`configForModifAS.yaml` (used by `dRNAas.py` and `dRNAmodif_2.py`) groups
samples under a `conditions` block, where each condition lists its replicate
sample names. It also separates the two references these modules need:

```yaml
gtf: "/path/to/annotation.gtf"      # dRNAas (suppa generateEvents)
fasta: "/path/to/transcriptome.fa"  # dRNAmodif_2 (nanocompore sampcomp)
conditions:
  condition1: ["S1R1", "S1R2"]
  condition2: ["S2R1", "S2R2"]
```

The rules expand automatically, so adding a condition or a replicate only
requires editing this block (no need to edit the `.py` files). Both diffSplice
and sampcomp compare exactly two conditions.

We also offer examples of config files in /examples/
